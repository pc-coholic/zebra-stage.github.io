Used to enter one or more custom rules by which to restrict USB peripheral usage when the device is in USB Host Mode. Allowed peripherals can be categorized by device class (sometimes referred to as device "type"), vendor ID (VID) and/or product ID (PID).

More than one rule can apply to the same peripheral. For example, there could be a rule to allow all peripherals of the "printer" class by specifying the appropriate numeric device class for printers (07h). Under this rule, all printers would be usable. There could be a separate rule to allow all "Zebra" peripherals by specifying the appropriate numeric vendor code for Zebra (0A5F). If those two rules existed at once, a Zebra printer would be usable under either rule whenever the device with the Custom Rules option selected is in USB Host Mode. If either of those rules were deleted, Zebra printers would remain usable. A peripheral is usable if there is at least one rule that allows it, and unusable if there are NO rules that allow it.

Example XML File

The XML code below 


<?xml version="1.0" encoding="UTF-8"?>

<usbconfig mode="whitelist">
  <class>AINPUT_SOURCE_DPAD</class>
  <class>AINPUT_SOURCE_JOYSTICK</class>
  <class>USB_CLASS_AUDIO</class>
    <vid vendorid="1504">
    <pid>5889</pid>
    </vid> 
</usbconfig>
