APIs

Mobile Parcel 1.0

Overview

Mobile Parcel API is designed based on the Android Intent architecture, reducing library dependencies and integration time for app developers. To ensure the interface is secure, it uses explicit and pending intents secured using an API token.

APIs available:

  • ENABLE_DIMENSION - Starts the dimensioning service and configures the time-of-flight (ToF) camera for dimensioning.
  • GET_DIMENSION - Presents the dimensioning UI layout on the app and sends dimensioning results back to the app.
  • GET_DIMENSION_PARAMETER - Reports the parameter values supported by the dimensioning framework.
  • SET_DIMENSION_PARAMETER - Sets parameters supported by the dimensioning framework.
  • DISABLE_DIMENSION - Releases all dimensioning framework resources, including camera hardware.

More information on each API are provided in the following sections.

Security

To ensure that only trusted applications communicate with the Dimensioning Framework, Zebra requires that developers use an access control method through Access Manager. For detailed instructions, refer to MX Access Manager.

The application must send both the API_TOKEN and the package name with each API request. If either the token or package name is incorrect, the Dimensioning API reports the result code Failed with a result message of Access Denied.

Requirements

Package Name

Add the <queries> tag in the AndroidManifest.xml of the app to define the Dimension Framework package name. This is required to allow the Mobile Parcel Dimensioning API to find the app.

<queries>
    <package android:name="com.zebra.dimensioning"/>
</queries>

Enable Dimension

ENABLE_DIMENSION starts the dimensioning service and enables and configures the time-of-flight (ToF) camera for the app to be ready for dimensioning. This must be called before any other API to ensure the dimensioning framework is enabled, allowing communication with the app.

NOTE: ENABLE_DIMENSION prevents other apps from accessing the ToF camera until DISABLE_DIMENSION is called. To save battery and reduce resource conflicts, Zebra recommends to call DISABLE_DIMENSION when dimensioning is not in use or when the app is running in the background.

Request

Use the following action to call ENABLE_DIMENSION:

com.zebra.dimensioning.ENABLE_DIMENSION

Get Dimension

GET_DIMENSION returns the values of the parcel measurements when an end-user dimensions a parcel. When calling this intent, the dimensioning framework renders its user interface (UI) on top of the application, allowing the user to aim the camera at the parcel and begin dimensioning. Upon successful dimensioning and confirmation from the user, the results are sent back to the application through an intent response. If dimensioning fails, the dimensioning results are set to read-only values and a result message displays details about the cause of failure.

NOTE: When calling GET_DIMENSION, dimensioning UI controls overlay on top of the application, preventing the user from accessing the application until the dimensioning is complete or the user exits the dimensioning session by tapping the Back button.

com.zebra.dimensioning.GET_DIMENSION

Set Dimension Parameter

SET_DIMENSION_PARAMETER configures the supported dimensioning keys (parameters). Set any number of parameters through a single call by adding them as intent extras.

Request

ENABLE_DIMENSION must be called before calling SET_DIMENSION_PARAMETER. Supported values are reported by the SUPPORTED_UNITS value in GET_DIMENSION_PARAMETER.

NOTE: Changing DIMENSIONING_UNIT may change the ready value.

Sample Code

The following sections provide code samples for developing a Mobile Parcel Dimensioning app.

Exception Handling

Zebra highly recommends to disable the Dimensioning Framework if the app crashes for any reason, safely closing the session and freeing all dimensioning resources. This provides data security between apps and saves power consumption of the Mobile Parcel Dimensioning solution.

The following code informs the dimensioning framework about app crashes and resources released:

  • Make the following change to AndroidManifest.xml:

    <application>
    android:name=".ApplicationDimensioningClient"
</application>

  • Make the following change to the JAVA file:

    public class ApplicationDimensioningClient extends Application {
    private static final String TAG = ApplicationDimensioningClient.class.getSimpleName();


    @Override
public void onCreate() {
    super.onCreate();
    Thread.setDefaultUncaughtExceptionHandler(new Thread.UncaughtExceptionHandler() {
        @Override
        public void uncaughtException(Thread thread, Throwable ex) {
            handleUncaughtException(thread, ex);
        } 
    });
}

public void handleUncaughtException (Thread thread, Throwable e) {
    Log.d(TAG, "handleUncaughtException start:");
    Intent intent = new Intent();
    intent.setAction(ConstantUtils.INTENT_ACTION_DISABLE_DIMENSION);
    intent.setPackage(ConstantUtils.SDK_PACKAGE);
    sendBroadcast(intent);
    Log.d(TAG, "handleUncaughtException stop:");
}

    

}

Service Unavailable

If the dimensioning service is not available, a notification can be sent to the app so it can initiate app data cleanup and inform the user of the unavailability of the dimensioning service.

The following code notifies the app when the dimensioning service is unavailable:

  • Add the following to AndroidManifest.xml:

    <application>
    android:name=".DimensioningSDKApp"
</application>

  • Make the following change to the JAVA file:

    public class DimensioningClientBroadcast extends BroadcastReceiver {
    private static final String TAG = DimensioningClientBroadcast.class.getSimpleName();


    @Override
public void onReceive(Context context, Intent intent) {
    Log.d(TAG, "onReceive() called.");
    String actionName = intent.getAction();
    if (actionName != null) {
        if (actionName.equals(ConstantUtils.INTENT_ACTION_SDK_APPLICATION_CRASH)) {
            Log.d(TAG, "onReceive() Dimensioning Service Unavailable");
            Toast.makeText(context.getApplicationContext(), String.valueOf(R.string.SDK_service_crashed), Toast.LENGTH_SHORT).show();
            Intent intentClean = new Intent (context, DimensioningClientApp.class);
            intentClean.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK | Intent.FLAG_ACTIVITY_CLEAR_TASK);
        } 
    }
}

    

}
