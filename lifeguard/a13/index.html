Migrating to Android 13

LifeGuard

Overview

Android 13 will present some potential challenges for organizations planning to migrate their Zebra SDM660 platform devices from earlier Android versions. The move requires conversion from full-disk to file-based encryption, which means that all device apps, data and settings must be erased from target devices. To simplify migration and preserve device data, Zebra is developing a unique process that can save up to 3GB of data from devices prior to migration, and automatically restore it afterward.

Part 1 of this guide covers migration scenarios in which data preservation is NOT required. If data preservation is required, skip to Part 2, which covers Zebra's non-destructive process.

Upgrade to A-13 (data loss)


 STOP!!
All device data is erased when upgrading SDM660 devices to Android 13 except data in the /enterprise partition.

Upgrading to Android 13 is supported only on Zebra TC21 devices at this time. Other models will follow as Android 13 becomes available. During the upgrade process, an Enterprise Reset is performed, removing all device data except that stored in the /enterprise partition. The process is otherwise identical to that of a traditional upgrade (shown below).

The following instructions are for Zebra StageNow, and should be modified accordingly if using an EMM system or Zebra OEMConfig.

To upgrade to A-13 without preserving device data:

1. Download the Android 13 OS image appropriate for the device:

  • TC21 devices with GMS
  • TC21 devices without GMS

2. Push the OS image to the device(s) by any available means (adb, StageNow, EMM, etc.).

3. Create a Profile with Power Manager to perform an OSUpdate, generate barcode(s) and scan with device scanner.

Success is indicated when device boots with Android 13.
Proceed with EMM enrollment and additional provisioning as desired.

Upgrade to A-13 (no data loss)

The process documented below allows for preservation of as much as 4GB of apps, data and settings on the device. To simplify the process and help preserve device data when migrating to Android 13, Zebra has developed Migration Magic LINK TO COME???, a utility that administrators can use to select and prioritize data to be preserved. File prioritization allows for a single "Golden Configuration" to be created for each device persona, allowing deploymemt of a single instruction set for preserving the MOST IMPORTANT data on devices that contain data in excess of the 4GB threshold.

Process Overview

1. Identify typical device persona(s) within the organization, which might include sets of devices with:

  • Specific Zebra-provided apps and data
  • Specific company-installed apps and data
  • System settings such as pre-configured Wi-Fi access points
  • EMM enrollment data and agent apps
  • Files and/or folders created by company or users

2. Use Zebra Migration Magic to create "Golden Configuration" for each device persona.

3. Use StageNow, OEMConfig or an EMM system to push the Golden Config to target devices.

4. Push the Android 13 OS image and special conversion package to the device(s).

5. Install the conversion package using Zebra Power Manager or an EMM system with the equivalent functionality.

The remainder of the process is completed automatically. Details of each part of the process are provided below.

Notes

The following facts apply to the Android 13 migration process.

  • Applies to SDM660-platform devices only (See SDM660-device list)
  • Converts device to file-based encryption
  • Preserves up to 4GB (+/-) of apps and data
  • Does NOT support Zebra's "Smart Profile" feature
  • Any data in /enterprise partition remains but is limited to ~250MB
  • Migration utility allows data prioritization for devices containing more than 4GB to preserve
  • MC2x and PS2x devices no longer share the same OS image package
  • GMS apps, data and settings are deleted
  • Expected availability: May, 2023

Limitations

The automated upgrade process is subject to the following limitations and restrictions.

  • All apps, data and settings not explicitly referenced in the "Golden Configuration" are deleted
  • Maximum backup size is 4GB (compressed)
  • The process clears any passwords, pins and passphrases
  • Failure to meet all preconditions (below) prior to update could result in upgrade failure (reverts to prior device state)
  • Devices with SD Cards formatted as Adoptable Storage are NOT supported
  • The process works only on the Primary User; work profiles, COPE and other multi-user scenarios are deleted
  • GMS apps, data and settings are NOT supported for backup
  • App(s) originally installed by an EMM system from Google Play and NOT backed up using the Zebra utility must be reinstalled manually after migration.
  • If an automated upgrade to fails (and remains booted in the interim OS), the upgrade to Android 13 must be completed manually; interim OS NOT suitable for production environments

Preconditions

The following conditions must be met on devices before attempting to upgrade them using this process.

  1. Device must boot to Android 11 with LifeGuard update NO LATER THAN FEB 2023 via fastboot or Full OTA packages. ???
  2. Must have >30% charge on device battery or be connected to AC power
  3. "Secure start" PIN/password feature DISABLED
    To set: Settings > Biometrics and security > Secure startup
  4. Screen lock disabled
    To set: Settings > Security > Screen Lock > PIN/Password/Pattern/Swipe
  5. Password Profile disabled
  6. SD Card must NOT be configured as Adoptable Storage
  7. Device encryption type must be "FDE"
    To set: Settings > About Phone > SW Components > Encryption Type > block
  8. USB Debugging MUST be ENABLED if logs are to be collected for debugging Also run RxLogger in case needed. ???

Zebra strongly recommends making compete backups of all devices before attempting migration.

I. Analyze Device Data

Create 'Golden Configuration'

On each device deemed typical of a given user persona:

Zebra's Migration Magic tool for Android automatically analyzes the device to visualize the apps and data to simplify the job of selecting and prioritizing what's most important to preserve after migration. The tool generates a list of all apps and data, presenting it with an interface that's easy to navigate and use to make selections and stay within the limit. Once the apps and data are identified for preservation, drag-and-drop apps and files to prioritize until the most important data is toward the top and the limit is not exceeded. When all files have been selected and prioritized for the persona, tap Proceed, then "Save XML" to store. This creates the "Golden Configuration" for this persona and stores it on the device. Copy the file from the device and deploy along with other migration files to devices of this persona.
