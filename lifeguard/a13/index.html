Migrating to Android 13

LifeGuard

Overview

To migrate SDM660-based devices—by far Zebra's post popular platform— to Android 13 requires conversion from full-disk to file-based encryption, which erases all device data. To simplify the process and preserve customer apps and data, Zebra offers Migration Magic ¿¿ WILL THERE BE A LINK ??, a simple Android utility that can preserve as much as 3GB of data from a device prior to migration, automatically restoring it afterward.

RE-DO DO-DO Run Migration Magic Using a device app provided by Zebra, select the most important device data to preserve and create "Golden Configuration" for each worker persona. Each persona can include as much as 3GB of apps, data and device settings information, and file prioritization allows for a single Golden Configuration to be created for each device persona, even if some devices contain more than 3GB of data.

Migration is performed in four parts:

  1. Use Migration Magic to select data to preserve and create "Golden Configuration" for each worker persona.
  2. Use StageNow, OEMConfig or an EMM to push the Golden Config to target devices.
  3. Push the Android 13 OS image and a special conversion package to the device(s).
  4. Install the conversion package using Zebra Power Manager or an EMM with the equivalent functionality.

The remainder of the process is completely automated. Details of each part of the process are provided below.

Notes

  • Applies to SDM660-platform devices only (See complete list.) ¿¿ IS THERE SUCH A LIST ??
  • Preserves up to 3GB (+/-) of apps and data
  • Converts device to file-based encryption
  • Zebra's "Smart Profile" feature NOT supported for migration of SDM660 devices to Android 13
  • Any data in /enterprise partition remains but is limited to ~250MB
  • Allows data prioritization to determine what MUST be kept on devices with more than 3GB to preserve
  • MC2x and PS2x devices no longer share the same OS image package
  • Expected availability in mid-April
