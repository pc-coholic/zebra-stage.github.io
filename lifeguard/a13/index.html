Migrating to Android 13

LifeGuard

Overview

Android 13 will present some potential challenges for organizations planning to migrate their Zebra SDM660 platform devices from earlier Android versions. The move requires conversion from full-disk to file-based encryption, which typically requires a device reset and loss of all data. However, Zebra is developing a unique process that can save about 3GB of data from devices prior to migration, and automatically restore it afterward.

The first section of this guide covers migration scenarios in which data preservation is NOT required. If data preservation is required, skip to the second section, which covers Zebra's non-destructive process.

Devices that comply with Common Criteria Certification can upgrade to Android 13 using the normal process with no risk of data loss.

UPDATED: June 12, 2023

Upgrade to A-13 (no data loss)

The process documented below allows for preservation of about 3GB (after compression) of apps, data and settings on the device. To simplify the process and help preserve device data when migrating to Android 13, Zebra has developed Zebra File Guardian LINK TO COME, a utility that administrators can use to select and prioritize data to be preserved. File prioritization allows for a single "Golden Configuration" to be created for each device persona, allowing deploymemt of a single instruction set for preserving the MOST IMPORTANT data on devices that contain data in excess of the maximum.

Devices that comply with Common Criteria Certification can upgrade to Android 13 using the normal process with no risk of data loss.

Process Overview

  1. Download Zebra Conversion Package, push to a device and reboot.
  2. File Guardian utility appears. Select files to be preserved and save backup as the "Golden Configuration" for that device class.
  3. Pull the Golden Config from the device to the host PC.
  4. Push Golden Config to a specific location on target device(s) being upgraded to Android 13.
  5. Download the desired Android image from Zebra, change file name and push to specific location on target device(s).
  6. Push the Zebra Conversion Package to the root level of target device(s) and perform "OS Upgrade" with Zebra Power Manager Action or equivilent EMM function.

The process continues automatically and boots to Android 13 when completed successfully.

Notes

  • Applies to SDM660-platform devices only (see SDM660-device list)
  • Converts device to file-based encryption (from disk-based)
  • Preserves about 3GB (+/-) of apps and data
  • Migration utility allows data prioritization for devices containing more than 3GB to preserve
  • Any data in /enterprise partition is preserved, but is limited to ~250MB
  • Does NOT support Zebra's "Smart Profile" feature
  • GMS data and settings are preserved; GMS apps are deleted
  • Expected availability: Late June, 2023

Limitations

  • All apps, data and settings not explicitly referenced in the "Golden Configuration" are deleted.
  • Maximum backup size is about 3GB (after compression).
  • Failure to meet all preconditions (below) prior to upgrade could result in upgrade failure (reverting device to its state prior to beginning the process).
  • Devices with SD Cards formatted as Adoptable Storage are NOT supported.
  • The process works only on the device's Primary User. Work profiles, COPE and other multi-user scenarios are deleted.
  • GMS data and settings are preserved; GMS apps are deleted.
  • App(s) originally installed by an EMM system from Google Play and NOT backed up using the Zebra utility and must be reinstalled manually after migration.
  • If an automated upgrade fails and the device remains booted in the Zebra Conversion Package (interim OS), the upgrade to Android 13 must be completed manually; the interim OS is NOT suitable for production environments.

Preconditions

The following conditions must be met on devices before attempting to upgrade using this process.

  1. Device must contain LifeGuard update NO LATER THAN FEB 2023 via fastboot or Full OTA packages. ???
  2. Must have 30% charge (or more) on device battery or be connected to AC power
  3. "Secure start" PIN/password feature DISABLED
    To set: Settings > Biometrics and security > Secure startup
  4. Screen lock disabled
    To set: Settings > Security > Screen Lock > PIN/Password/Pattern/Swipe
  5. Password Profile disabled
  6. SD Card must NOT be configured as Adoptable Storage
  7. Device encryption type must be "FDE"
    To set: Settings > About Phone > SW Components > Encryption Type > block
  8. USB Debugging MUST be ENABLED if logs are to be collected for debugging Also run RxLogger in case needed. ???

Zebra strongly recommends backing up devices before attempting migration.

Process Details

To upgrade Zebra SDM660-platform devices to Android 13:

1. Identify typical device persona(s) within the organization, which might include sets of devices with:

  • Specific Zebra-provided apps and data
  • Specific company-installed apps and data
  • System settings such as pre-configured Wi-Fi access points, passwords, etc.
  • EMM enrollment data and agent apps
  • Files and/or folders created by the company or device users

2. Download the Zebra Conversion Package (ZCP) LINK TO COME, push it to the root level of one such device and reboot.

3. At boot, the device automatically runs File Guardian, a Zebra utility used to select apps and data to be backed up and restored to devices after migration. The resulting backup is called a "Golden Configuration."

When finished, tap "Proceed" and then "Save XML" to store the Golden Config:

e. When this message is displayed, it's safe to proceed to the next step:

4. Get Golden Config from device at /enterprise/usr/persist/backup_restore_config.xml and store it on the administrative host PC.

5. Use StageNow, OEMConfig, an EMM system to push the Golden Config to target device(s) in the same location:

  • /enterprise/usr/persist/backup_restore_config.xml

6. Download the desired Android 13 OS image appropriate for the device:

  • TC21 devices with GMS
  • TC21 devices without GMS

Reminder: This process is for Zebra SDM660-platform devices only. Other platforms can use the Quick Upgrade process with no fear of data loss.

7. Push the Android 13 OS image to the device(s) using the following path and file name:

  • /data/tmp/public/FBE_FOTA.zip
    The A13 image MUST be in this location with this file name or the process will FAIL.

8. Push the Zebra Conversion Package to target device(s) to be upgraded to Android 13.

9. Perform an "OS Upgrade" on the ZCP using Zebra MX Power Manager (or the equivalent EMM function).

The remainder of the process is automatic. Success is confirmed when device(s) boot to Android 13.

A-13 Quick Upgrade (data loss)


 STOP!!
All device data is erased when upgrading SDM660 devices to Android 13 except data in the /enterprise partition.

Upgrading to Android 13 is supported only on Zebra TC21 devices at this time. Other models will follow as Android 13 becomes available. During the upgrade process, an Enterprise Reset is performed, removing all device data except that stored in the /enterprise partition. The process is otherwise identical to that of a traditional upgrade (shown below).

The following instructions are for Zebra StageNow, and should be modified accordingly for use with Zebra OEMConfig or a company's own an EMM system.

To upgrade to A-13 without preserving device data:

1. Download the Android 13 OS image appropriate for the device:

  • TC21 devices with GMS
  • TC21 devices without GMS

2. Push the OS image to the device(s) by any available means (adb, StageNow, EMM, etc.).

3. Create a Profile with Power Manager to perform an OSUpdate, generate barcode(s) and scan with device scanner.

Success is indicated when device boots with Android 13.
Proceed with EMM enrollment and additional provisioning as desired.

Notes

  • LifeGuard Over the Air is not supported for Android 13 OS image deployment (coming later in 2023)
  • Zebra has successfully tested persistence only when using Zebra Persist Manager
  • Persistence by means other than Zebra PersistMgr cannot be guaranteed
  • Upgrades to Android 13 are supported only from devices running Android 8, 9, 10 or 11
