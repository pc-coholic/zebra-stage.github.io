APIs

Mobile Parcel Dimensioning 1.0

Overview

Mobile Parcel Dimensioning API is designed based on the Android Intent architecture, reducing library dependencies and integration time for app developers. To ensure the interface is secure, the Mobile Parcel Dimensioning API uses explicit and pending intents secured using an API token.

APIs available:

  • ENABLE_DIMENSION - Opens the dimensioning service and configures the time-of-flight (ToF) camera for dimensioning.
  • GET_DIMENSION - Presents the dimensioning UI layout on the line-of-business (LOB) app and sends dimensioning results back to the LOB app.
  • GET_DIMENSION_PARAMETER - Reports the parameter values supported by the dimensioning framework.
  • SET_DIMENSION_PARAMETER - Sets parameters supported by the dimensioning framework.
  • DISABLE_DIMENSION - Releases all framework resources, including camera hardware.

The sections that follow provide details on each API.

Security

To ensure that only trusted applications communicate with the Dimensioning Framework, Zebra requires that developers use an access control method through Access Manager. For detailed instructions, refer to MX Access Manager.

The application must send both the API_TOKEN and the package name with each API request. If either the token or package name is incorrect, the Dimensioning API reports the result code Failed with a result message of Access Denied.

Enable Dimension

Call ENABLE_DIMENSION before any other API to ensure the framework is enabled, allowing communication with the LOB app. This API ensures that the DimensioningService and ToF camera are enabled, configured, and ready for dimensioning.

NOTE: Calling ENABLE_DIMENSION prevents other apps from accessing the ToF camera until DISABLE_DIMENSION is called. To save battery and reduce resource conflicts, it is recommended to call DISABLE_DIMENSION when dimensioning is not in use or when the LOB app is running in the background.

com.zebra.dimensioning.ENABLE_DIMENSION

Get Dimension

Call GET_DIMENSION when the end-user is ready to begin dimensioning a parcel. On calling this intent, the dimensioning framework renders the user interface (UI) on top of the application, allowing the user to aim the camera at the parcel and begin dimensioning. On successful dimensioning and confirmation from the user, the results are sent back to the LOB application through an intent response. On failure, dimensioning results are set to read-only values, and a result message displays details about the cause of the failure.

NOTE: When calling this API, application controls are hidden behind the dimensioning UI controls. The user is not able to control the application until the dimensioning is complete or the user exits the dimensioning session by selecting the Back button.

Set Dimension Parameter

Use SET_DIMENSION_PARAMETER API to configure the supported dimensioning keys (parameters). Set any number of parameters through a single call by adding them as intent extras.

Set Dimension Parameter Request

Make sure to call ENABLE_DIMENSION before calling SET_DIMENSION_PARAMETER. Supported values are reported by the SUPPORTED_UNITS value in GET_DIMENSION_PARAMETER.

NOTE: Changing DIMENSIONING_UNIT may change the ready value.

Sample Code

The following sections provide code samples for developing the LOB app.

Package Name

Use the <queries> tag in the AndroidManifest.xml to define the Dimension Framework package name. This is required to allow the SDK to find the LOB app.

<queries>
    <package android:name="com.zebra.dimensioning"/>
</queries>

Exception Handling from App

The following code informs the Dimension Framework about app crashes and resources released. It is highly recommended to disable the Dimensioning Framework if the LoB app crashes for any reason, safely closing the session and freeing all resources. This provides data security between apps and saves power consumption from the Mobile Parcel Dimensioning solution. Make the following change to AndroidManifest.xml.

<application>
    android:name=".ApplicationDimensioningClient"
</application>

Make the following change to the JAVA file:

public class ApplicationDimensioningClient extends Application {
    private static final String TAG = ApplicationDimensioningClient.class.getSimpleName();

    @Override
    public void onCreate() {
        super.onCreate();
        Thread.setDefaultUncaughtExceptionHandler(new Thread.UncaughtExceptionHandler() {
            @Override
            public void uncaughtException(Thread thread, Throwable ex) {
                handleUncaughtException(thread, ex);
            } 
        });
    }

    public void handleUncaughtException (Thread thread, Throwable e) {
        Log.d(TAG, "handleUncaughtException start:");
        Intent intent = new Intent();
        intent.setAction(ConstantUtils.INTENT_ACTION_DISABLE_DIMENSION);
        intent.setPackage(ConstantUtils.SDK_PACKAGE);
        sendBroadcast(intent);
        Log.d(TAG, "handleUncaughtException stop:");
    }
}

Notify App About Service Crash

The following code for the dimensioning service informs the LOB app about a service crash. This helps the app initiate app data cleanup and inform the user that the dimensioning service is not available.

Make the following change to AndroidManifest.xml:

<application>
    android:name=".DimensioningSDKApp"
</application>

Make the following change to the JAVA file:

public class DimensioningClientBroadcast extends BroadcastReceiver {
    private static final String TAG = DimensioningClientBroadcast.class.getSimpleName();

    @Override
    public void onReceive(Context context, Intent intent) {
        Log.d(TAG, "onReceive() called.");
        String actionName = intent.getAction();
        if (actionName != null) {
            if (actionName.equals(ConstantUtils.INTENT_ACTION_SDK_APPLICATION_CRASH)) {
                Log.d(TAG, "onReceive() SDK crash");
                Toast.makeText(context.getApplicationContext(), String.valueOf(R.string.SDK_service_crashed), Toast.LENGTH_SHORT).show();
                Intent intentClean = new Intent (context, DimensioningClientApp.class);
                intentClean.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK | Intent.FLAG_ACTIVITY_CLEAR_TASK);
            } 
        }
    }
}
